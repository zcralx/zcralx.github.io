Stream流式编程

了解Stream

  • Stream 是 Java8 添加的一个新的抽象称为流，可以让你以一种声明的方式处理数据。通过使用流，我们可以说明想要完成什么任务，而不是说明如何去实现它。我们将操作的调度留给具体实现去解决。例如，假设我们想要计算某个属性的平均值，那么我们就可以指定数据源和该属性，然后，流库就可以对计算进行优化，例如，使用多线程来计算总和与个数，并将结果合并。
  • Stream 可以极大提高Java程序员的生产力，让程序员写出高效率、干净、简洁的代码。
  • Stream 流的操作是以管道的方式串起来的。管道以数据源开始，包含若干个中间操作，最终以终点操作结束(规约操作）。
  • 流并不存储其元素。这些元素可能存储在底层的集合中，或者是按需生成的。
  • 流的操作不会修改其数据源。例如，filter 方法不会从流中移除元素，而是会生成一 个新的流，其中不包含被过滤掉的元素。
  • 流的操作是尽可能惰性执行的。这意味着直至需要其结果时，操作才会执行。因此，我们甚至可以操作无限流。

操作流的典型流程

建立一个包含三个阶段的操作管道：

  1. 创建一个流
  2. 指定将初始流转换为其他流的中间操作，可能有多个中间操作
  3. 归约操作，产生结果，此操作会强制执行之前的惰性操作，然后这个流就再也无法使用了

创建流

  1. 使用Collection接口的stream方法将任何集合转换为流：

    default Stream<E> stream() {
        return StreamSupport.stream(spliterator(), false);
}

    spliterator是一个实际的 Spliterator 对象，它代表了可以被分割和遍历的数据源; false表示是否应该创建一个并行流（parallel stream）。传入的是 false，意味着创建的是一个顺序流，这意味着流的操作将在当前线程上按照元素的自然顺序逐个进行处理。

    List<String> list = Arrays.asList("1ded","2cec","2dc","4vv","4vrv");
list.stream().filter(s -> s.startsWith("2")).forEach(System.out::println);
//2cec    2dc

    关于顺序流可以参考下面的例子：

    public class StreamTest {
    private static final List<Apple> appleStore = Arrays.asList(
            new Apple(1, "red", 500, "湖南"),  //id1
            new Apple(2, "red", 100, "天津"),  //id2
            new Apple(3, "green", 300, "湖南"), //id3
            new Apple(4, "green", 200, "天津"),  //id4
            new Apple(5, "green", 100, "湖南")  //id5
    );
    public static void main(String[] args) {
        appleStore.stream().filter(apple -> apple.getWeight() > 100)
                .peek(apple -> System.out.println("通过第1层筛选 " + apple))
                .filter(apple -> "green".equals(apple.getColor()))
                .peek(apple -> System.out.println("通过第2层筛选 " + apple))
                .filter(apple -> "湖南".equals(apple.getBirthplace()))
                .peek(apple -> System.out.println("通过第3层筛选 " + apple))
                .collect(Collectors.toList());
    }
}

    它的执行过程是这样的：id1先过滤，通过了第一层重量筛选但是颜色不对没通过第二层；换id2过滤，重量都没通过；换id3过滤，三层都通过了……

    输出结果：

    通过第1层筛选 com.example.javalearn.rui5.Apple@7cca494b
通过第1层筛选 com.example.javalearn.rui5.Apple@7ba4f24f
通过第2层筛选 com.example.javalearn.rui5.Apple@7ba4f24f
通过第3层筛选 com.example.javalearn.rui5.Apple@7ba4f24f
通过第1层筛选 com.example.javalearn.rui5.Apple@3b9a45b3
通过第2层筛选 com.example.javalearn.rui5.Apple@3b9a45b3

创建无限流

Stream接口有两个创建无限流的静态方法：

  1. public static<T> Stream<T> generate(Supplier<? extends T> s/*用于提供Stream中的元素,反复调用函数s*/) {
        Objects.requireNonNull(s);//确保传入的Supplier不为空
				return StreamSupport.stream(
new StreamSpliterators.InfiniteSupplyingSpliterator.OfRef<>(Long.MAX_VALUE, s)/*用于创建一个无限的SupplyingSpliterator，其中s用于提供元素。*/, false);
}

int[] count = {0};
// 创建一个无限递增的整数流
Stream<Integer> infiniteIntegers = Stream.generate(() -> count[0]++);
// 取前10个整数打印
infiniteIntegers.limit(10).forEach(System.out::println);

    无论何时，只要需要 一个流类型的值，该函数就会被调用以产生一个这样的值。

  2. public static<T> Stream<T> iterate(final T seed, final UnaryOperator<T> f) {
        Objects.requireNonNull(f);
Spliterator<T> spliterator = new Spliterators.AbstractSpliterator<>(Long.MAX_VALUE,Spliterator.ORDERED | Spliterator.IMMUTABLE) {/*匿名类*/					
  					T prev;
						boolean started;
//Spliterator实现了tryAdvance方法，每次调用该方法时，会将序列中的下一个元素传递给给定的Consumer行动，并返回true。
            @Override
            public boolean tryAdvance(Consumer<? super T> action) {
                Objects.requireNonNull(action);
                T t;
                if (started)
                    t = f.apply(prev);
                else {
                    t = seed;
                    started = true;
                }
                action.accept(prev = t);
                return true;
            }
        };
        return StreamSupport.stream(spliterator, false);
}

    如何利用iterate方法产生一个有限的序列呢？添加一个谓词（Predicate）来描述迭代应该如何结束

    static<T> Stream<T> iterate(T seed, Predicate<? super T> hasNext, UnaryOperator<T> next)

var limit = new BigInteger("10000");
Stream<BigInteger> integers = Stream.iterate(BigInteger.ZERO, 
n ->n.compareTo(limit) < 0, n -> n. add(BigInteger. ONE));
//该序列中的第一个元素是种子BigInteger.ZERO，第 二个元素是f(seed)，即1(作为大整数)，下一个元素是f(f(seed)，即2，后续以此类推。   只要该谓词拒绝了某个选代生成的值，这个流即结束

流的转换

流的转换会产生一个新的流，它的元素派生自另一个流中的元素。常用的有filter,map和flatMap

Stream<T> filter (Predicate<? super T> predicate)

产生一个流，它包含当前流中所有满足谓词条件的元素。可以参考上面的例子。

<R> Stream<R> map(Function<? super T，? extends R> mapper)

产生一个流，它包含将mapper应用于当前流中所有元素所产生的结果

带方法引用的map：

List<String> list = Arrays.asList("1ded","2cec","2dc","4vv","4vrv");

Stream<String> uppercase = list.stream().map(String::toUpperCase);
uppercase.forEach(System.out::println);//1DED 2CEC......
或者这样写：
list.stream().map(String::toUpperCase).forEach(System.out::println);
//因为forEach方法没有返回值，是一个void

lambda表达式的map：

list.stream().map(s -> s.substring(0,1)).forEach(System.out::println);//1 2 2 4 4

flatMap方法接受一个lambda表达式函数， 函数的返回值必须也是一个stream类型，flatMap方法最终会把所有返回的stream合并，map方法做不到这一点

<R> Streams<R> flatMap(Function <? super T, ? extends Stream<? extends R>> mapper)

通过一个案例来理解：对给定单词列表 [“Hello”,“World”],你想返回列表[“H”,“e”,“l”,“o”,“W”,“r”,“d”]

使用map：

String[] words = new String[]{"Hello","World"};
List<String[]> a = Arrays.stream(words)
                .map(word -> word.split(""))
                .distinct()
                .collect(toList());
a.forEach(System.out::print);
//[Ljava.lang.String;@6d311334[Ljava.lang.String;@682a0b20返回了两个地址

这个实现方式是有问题的，传递给map方法的lambda为每个单词生成了一个String[]（String列表）。因此，map返回的流实际上是Stream<String[]> 类型的。你真正想要的是用Stream来表示一个字符串。

使用flatMap：

List<String> a = Arrays.stream(words)
                .map(word -> word.split(""))
                .flatMap(Arrays::stream)
                .distinct()
                .collect(toList());
a.forEach(System.out::print);//HeloWrd
